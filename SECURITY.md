# Security Policy

## Supported Versions

Security fixes are applied to the latest stable Dashchan_2 release. Older releases and beta builds may be used to reproduce a problem but are not maintained as separate supported lines.

## Reporting A Vulnerability

Do not open a public issue for a vulnerability that could expose users, credentials, posting identities, update delivery, file access, or signing material.

Send a report to `dashchan_2@mail.ru` containing:

- affected version and Android version;
- clear reproduction steps;
- expected security impact;
- relevant code locations or a minimal proof of concept;
- whether the issue has been disclosed elsewhere.

Remove unrelated personal data from logs and attachments. Never send real Passcodes, cookies, private keys, or account credentials.

The project is maintained on a best-effort basis. Receipt will be acknowledged when possible, the report will be assessed, and publication should wait until users have a reasonable opportunity to update.

## Release Verification

Install APKs from the project's [GitHub Releases](https://github.com/andrewpozdnakov7-jpg/Dashchan_2/releases). Certificate verification guidance is available in [docs/SIGNING.md](docs/SIGNING.md).
